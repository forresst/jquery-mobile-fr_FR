Structure d'entête

L'entête est une barre d'outils en haut de la page qui contient généralement le texte du titre de page et des boutons optionnels placés à la gauche et/ou à la droite du titre pour la navigation ou pour des actions.

Le texte du titre est normalement un élément de titre H1 mais c'est possible d'utiliser n'importe quel niveau de titre (H1-H6) pour permettre une flexibilité sémantique. Par exemple, une page contenant plusieurs 'pages' mobile peut utiliser un élément H1 sur 'page' d'accueil et un élément H2 sur les pages secondaires. Tous les niveaux de titre sont de style identique par défaut pour maintenir une cohérence visuelle.

<div data-role="header"> 
	<h1>Titre de page</h1> 
</div>

Caractéristiques de l'entête par défaut

La barre d'outils d'entête sera thématisé avec le nuancier "a" par défaut (noir dans le thème par défaut) mais vous pouvez facilement définir une couleur de nuancier du thème.

Ajout de boutons

Dans la configuration standard de l'entête, il y a des emplacements pour les boutons de chaque côté du texte du titre. Chaque bouton est généralement un élément anchor, mais toutes les balises de bouton valide fonctionneront. Pour économiser de l'espace, les boutons dans les barres d'outils sont définis avec un style aligné de sorte que le bouton soit aussi large que le texte et les icônes qu'il contient.

Positionnement par défaut des boutons

Le plugin d'entête regarde les enfants immédiats du conteneur d'entête, et configure automatiquement le premier lien dans le bouton de gauche et le deuxième lien dans le droit. Dans cet exemple, le bouton 'Annuler' apparaîtra dans l'emplacement de gauche et 'Enregistrer' apparaîtra dans l'emplacement de droite en fonction de leur séquence dans l'ordre de la source. 

			
<div data-role="header" data-position="inline">
	<a href="index.html" data-icon="delete">Annuler</a>
	<h1>Modifier le contact</h1>
	<a href="index.html" data-icon="check">Enregistrer</a>
</div>

Les boutons adoptent automatiquement la couleur du nuancier de la barre où ils sont positionnés, de sorte qu'un lien dans une barre d'entête avec la couleur "a" sera également stylisé comme les boutons colorés "a". C'est simple de faire un bouton qui ressort visuellement - ici, nous ajoutons l'attribut data-theme et définissons le nuancier de couleur pour le bouton à "b" pour mettre en évidence le bouton "Enregistrer".

			
<div data-role="header" data-position="inline">
	<a href="index.html" data-icon="delete">Annuler</a>
	<h1>Modifier le contact</h1>
	<a href="index.html" data-icon="check" data-theme="b">Enregistrer</a>
</div>

Contrôler le positionnement du bouton avec des classes

La position du bouton peut également être contrôlée par l'ajout de classes à l'ancrage du bouton, plutôt que de compter sur l'ordre de la source. Ceci est particulièrement utile si vous voulez seulement un bouton dans l'emplacement de droite. Pour spécifier la position du bouton, ajoutez la classe ui-btn-left ou ui-btn-right à l'ancrage.


<div data-role="header" data-position="inline"> 
	<h1>Titre de la page</h1>
	<a href="index.html" data-icon="gear" class="ui-btn-right">Options</a>
</div>

Ajout des boutons Retour

jQuery Mobile a une fonctionnalité pour automatiquement créer et d'ajouter des boutons "retour" pour l'entête, bien qu'elle soit désactivée par défaut. Ceci est principalement utile dans les applications installées sans habillage, tels que celles qui fonctionnent dans une vue web app native. Le framework génère automatiquement un bouton "retour" sur une entête lorsque l'option addBackBtn du plugin page est à true. Cela peut aussi être définie via le balisage si le div de la page a un attribut data-add-back-btn="true".

Si vous utilisez l'attribut data-rel="back" sur une ancre, tout clic sur cette ancre imitera le bouton retour, remontant une entrée de l'historique et en ignorant le href par défaut de l'ancre. Ceci est particulièrement utile lors de la liaison de retour vers une page nommée, comme un lien qui s'appelle "accueil", ou lors de la génération des boutons "retour" avec JavaScript, comme un bouton pour fermer une fenêtre de dialogue. Lorsque vous utilisez cette fonctionnalité dans votre balisage source, assurez vous de fournir un href significatif qui pointe réellement sur l'URL de la page de référence (cela permettra à la fonction de travailler pour les utilisateurs de C-Grade navigateurs). Aussi, s'il vous plaît garder à l'esprit que si vous voulez juste une transition inverse sans réellement retourner dans l'historique, vous devez utiliser à la place l'attribut data-direction="reverse".

Personnalisation du texte du bouton Retour

Si vous souhaitez configurer le texte du bouton retour, vous pouvez soit utiliser l'attribut data-back-btn-text="précédent" sur ​​l'élément de votre page, ou le définir par programmation via les options du plugin de page :
$.mobile.page.prototype.options.backBtnText = "précédent";

Style par défaut du bouton Retour

Si vous souhaitez configurer le thème du bouton Retour, vous pouvez utiliser :
$.mobile.page.prototype.options.backBtnTheme = "a";
Si vous faites cela par programmation, définissez cette option à l'intérieur du gestionnaire d'événements mobileinit.

Configurations personnalisées de l'entête

Si vous avez besoin de créer une entête qui ne suit pas la configuration par défaut, il suffit d'envelopper votre balisage personnalisé de style dans un conteneur, comme un div. Le plugin n'appliquera pas automatiquement la logique du bouton pour le contenu enveloppé dans l'entête de sorte que vous puissiez écrire des styles personnalisés pour la mise en forme du contenu de votre entête.

Il est également possible de créer des barres personnalisées sans jamais utiliser le data-role de l'entête. Par exemple, commencez avec n'importe quel conteneur et ajoutez la classe ui-bar pour demander un espacement standard de la barre et ajoutez la classe ui-bar-b pour attribuer les styles du nuancier de la barre à partir de votre thème (le "b" peut être n'importe quelle lettre du nuancier).


<div class="ui-bar ui-bar-b">
	<h3>Je suis juste un div avec des classes de barre et un <a href="#" data-role="button">Bouton</a></h3>
</div>

Notez que .ui-bar ne doit pas être ajouté à l'entête ou au pied de page des barres qui s'étendent sur toute la largeur de la page, car l'espacement supplémentaire entraînera un élément sur ​​toute la largeur qui sortira de son conteneur parent. Pour ajouter de l'espace à l'intérieur d'une barre d'outils sur toute la largeur il est recommandé d'envelopper le contenu de la barre d'outils dans un élément, et d'appliquer l'espacement à cet élément.
