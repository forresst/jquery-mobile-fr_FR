Apparence d'une liste

Le plugin listview a les options suivantes :

countTheme string

Par défaut : "c"

Définit le jeu de couleurs (nuancier) pour les éléments bulles compteur de la liste. Il accepte une seule lettre de A à Z qui correspond aux nuanciers inclus dans votre thème. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.listview.prototype.options.countTheme = "a";
});

Cette option est également définie comme un attribut de données : data-count-theme="a".

dividerTheme string

Par défaut : "b"

Définit le jeu de couleurs (nuancier) pour les séparateurs de liste. Il accepte une seule lettre de A à Z qui correspond aux nuanciers inclus dans votre thème. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.listview.prototype.options.dividerTheme = "a";
});

Cette option est également définie comme un attribut de données : data-divider-theme="a".

filter boolean

par défaut : false

Ajoutez une barre de recherche aux listes. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.listview.prototype.options.filter = true;
});

Cette option est également définie comme un attribut de données : data-filter="true".

filterCallback function

La fonction pour déterminer quels éléments de la liste sont à masquer lorsque la zone de saisie du filtre de recherche change. La fonction accepte deux arguments -- le texte de la liste d'éléments (ou la valeur data-filtertext si présente) et la chaine de recherche. Retourne true pour masquer l'élément, false pour le laisser visible. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.listview.prototype.options.filterCallback = function( text, searchValue ) {
        // Seuls les éléments affichés sont ceux qui *commence* avec la chaine de recherche
        return text.toLowerCase().substring( 0, searchValue.length ) !== searchValue;
    };
});
filterPlaceholder string

Par défaut : "Filter items..."

Le texte du placeholder utilisé dans les barres de recherche. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.listview.prototype.options.filterPlaceholder = "Rechercher...";
});

Cette option est également définie comme un attribut de données : data-filter-placeholder="Rechercher...".

filterTheme string

Par défaut : "c"

Définit le jeu de couleurs (nuancier) pour la barre de recherche. Il accepte une seule lettre de A à Z qui correspond aux nuanciers inclus dans votre thème. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.listview.prototype.options.filterTheme = "a";
});

Cette option est également définie comme un attribut de données : data-filter-theme="a".

headerTheme string

Par défaut : "b"

Définit le jeu de couleurs (nuancier) pour les entêtes de liste intégrée des sous pages. Il accepte une seule lettre de A à Z qui correspond aux nuanciers inclus dans votre thème. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.listview.prototype.options.headerTheme = "a";
});

Cette option est également définie comme un attribut de données : data-header-theme="a".

initSelector CSS selector string

Par défaut: ":jqmData(role='listview')"

Ceci est utilisé pour définir les sélecteurs (types d'éléments, les data roles, etc) qui seront automatiquement initialisés comme des listes. Pour changer les éléments qui sont initialisés, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.listview.prototype.options.initSelector = ".mylistview";
});
inset boolean

par défaut : false

Ajouter les styles de liste intégrée styles. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.listview.prototype.options.inset = true;
});

Cette option est également définie comme un attribut de données : data-inset="true".

splitIcon string

Par défaut : "arrow-r"

Applique une icône de l'ensemble d'icônes à tous les boutons de liste. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.listview.prototype.options.splitIcon = "star";
});

Cette option est également définie comme un attribut de données : data-split-icon="star".

splitTheme string

Par défaut : "b"

Définit le jeu de couleurs (nuancier) pour les boutons de liste. Il accepte une seule lettre de A à Z qui correspond aux nuanciers inclus dans votre thème. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.listview.prototype.options.splitTheme = "a";
});

Cette option est également définie comme un attribut de données : data-split-theme="a".

theme string

par défaut : null, hérité d'un parent

Définit le jeu de couleurs (nuancier) pour ce widget. Il accepte une seule lettre de A à Z qui correspond aux nuanciers inclus dans votre thème. Par défaut, il hérite de la même couleur de nuancier que le conteneur parent qui le possède s'il n'est pas explicitement défini. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.listview.prototype.options.theme = "a";
});

Cette option est également définie comme un attribut de données : data-theme="a".
