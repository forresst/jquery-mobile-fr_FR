Le plugin ensemble de pliable a les méthodes suivantes :

collapsedIcon string

Par défaut : "plus"

Définit l'icône pour les conteneurs des entêtes des pliables dans l'état plié. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).on( "mobileinit", function(){
    $.mobile.collapsibleset.prototype.options.collapsedIcon = "arrow-r";
});

Cette option est également définie comme un attribut de données : data-collapsed-icon="arrow-r".

corners boolean

Par défaut : true

Applique border-radius de thème au premier et dernier pliable si défini à true. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).on( "mobileinit", function(){
    $.mobile.collapsible.prototype.options.corners = false;
});

Cette option est également définie comme un attribut de données : data-corners="false".

expandedIcon string

par défaut : "minus"

Définit l'icône pour les conteneurs des entêtes des pliables dans l'état déplié. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).on( "mobileinit", function(){
    $.mobile.collapsibleset.prototype.options.expandedIcon = "arrow-d";
});

Cette option est également définie comme un attribut de données : data-expanded-icon="arrow-d".

iconpos string

par défaut : "left"

Positions des icônes dans les entêtes du pliable. Les valeurs possibles : left, right, top, bottom, none, notext.

$( document ).on( "mobileinit", function(){
    $.mobile.collapsibleset.prototype.options.iconpos = "right";
});

Cette option est également définie comme un attribut de données : data-iconpos="right".

initSelector CSS selector string

default: ":jqmData(role='collapsible-set')"

Ceci est utilisé pour définir les sélecteurs (types d'éléments, les data roles, etc) qui seront automatiquement initialisés comme des ensembles de pliable. Pour changer les éléments qui sont initialisés, liez cette option à l'événement mobileinit :

$( document ).on( "mobileinit", function(){
    $.mobile.collapsibleset.prototype.options.initSelector = ".mycollapsibleset";
});
inset boolean

Par défaut : true

En mettant cette option à false, les pliables auront une apparence sur toute la largeur sans coins. Si la valeur est false pour un conteneur individuel de pliable, mais que ce conteneur fait partie d'un ensemble pliable, alors la valeur est héritée de l'ensemble du parent pliable. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).on( "mobileinit", function(){
    $.mobile.collapsible.prototype.options.inset = false;
});

Cette option est également définie comme un attribut de données : data-inset="false".

mini boolean

par défaut : false

Définit la taille des pliables pour qu'ils soient plus compacts : une version mini. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).on( "mobileinit", function(){
    $.mobile.collapsibleset.prototype.options.mini = true;
});

Cette option est également définie comme un attribut de données : data-mini="true".

theme string

par défaut : null, hérité d'un parent

Définit le jeu de couleurs (nuancier) pour l'ensemble de pliable. Il accepte une seule lettre de a à z qui correspond aux nuanciers inclus dans votre thème. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).on( "mobileinit", function(){
    $.mobile.collapsibleset.prototype.options.theme = "a";
});

Cette option est également définie comme un attribut de données : data-theme="a".
