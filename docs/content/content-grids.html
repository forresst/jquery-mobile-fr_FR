Grille de mise en page

Accueil Recherche

Utiliser des mises en page de plusieurs colonnes, cela n'est généralement pas recommandé sur un appareil mobile en raison d'une largeur d'écran étroite, mais il y a des moments où vous pouvez avoir besoin de placer de petits éléments côte à côte (comme des boutons ou des onglets de navigation, par exemple).

Le framework jQuery Mobile fournit un moyen simple de construire des colonnes basé CSS à travers une convention de classe appelée ui-grid.

Il y a quatre mises en page de configurations prédéfinies qui peuvent être utilisés dans toutes les situations qui exigent des colonnes :

  • deux-colonnes (En utilisant la classe ui-grid-a)
  • trois-colonnes (En utilisant la classe ui-grid-b)
  • quatre-colonnes (En utilisant la classe ui-grid-c)
  • cinq-colonnes (En utilisant la classe ui-grid-d)

Les grilles sont de largeur 100%, totalement invisible (pas de bordure ou de fond d'écran) et n'ont pas d'espacement ou de marge, donc ils ne devraient pas interférer avec les styles des éléments placés à l'intérieur.

Dans le conteneur de grille, les éléments enfants sont affectés ui-block-a/b/c/d/e d'une manière séquentielle qui fait que chaque élément de "bloc" se positionne côte à côte, formant la grille. La classe ui-bloc-a efface essentiellement les positions qui vont commencer une nouvelle ligne (voir les grilles de plusieurs lignes, ci-dessous).

Grilles deux colonnes

Pour construire une mise en page à deux colonnes (50/50%), commencez avec un conteneur avec class de ui-grid-a. Et ajoutez deux conteneurs enfant à l'intérieur classé avec ui-block-a pour la première colonne et ui-block-b pour la seconde :


<div class="ui-grid-a">
	<div class="ui-block-a"><strong>Je suis le bloc A</strong> et le texte à l'intérieur sera envelopper</div>
	<div class="ui-block-b"><strong>Je suis le bloc B</strong> et le texte à l'intérieur sera envelopper</div>
</div><!-- /grid-a -->

Le balisage ci-dessus produit la disposition de contenu suivants :

Je suis le bloc A et le texte à l'intérieur sera envelopper.
Je suis le bloc B et le texte à l'intérieur sera envelopper.

Comme vous le voyez ci-dessus, les blocs de la grille par défaut n'ont pas de style visuel, ils présentent simplement le contenu côte à côte.

Les classes de la grille peuvent être appliqué à n'importe quel conteneur. Dans le prochain exemple, nous ajoutons ui-grid-a à fieldset et appliquons les classes ui-block aux conteneurs de chaque boutons pour les étirer chacun à 50% de la largeur de l'écran :


<fieldset class="ui-grid-a">
	<div class="ui-block-a"><button type="submit" data-theme="c">Annuler</button></div>
	<div class="ui-block-b"><button type="submit" data-theme="b">Valider</button></div>	   
</fieldset>

Veuillez noter que le framework ajoute la marge à gauche et à droite des boutons dans une grille. Pour un unique bouton, vous pouvez utiliser un conteneur avec la classe ui-grid-solo et enveloppez le bouton dans un div avec la classe ui-block-a Comme pour l'exemple ci-dessous. De cette façon, le bouton recevra la même marge.


<div class="ui-grid-a">
	<div class="ui-block-a"><button type="button" data-theme="c">Précédent</button></div>
	<div class="ui-block-b"><button type="button" data-theme="c">Suivant</button></div>	   
</div>
<div class="ui-grid-solo">
	<div class="ui-block-a"><button type="v" data-theme="b">Plus</button></div>
</div>

Les classes de thème (pas les attributs data-theme) du système de thématisation peuvent être ajoutées à un élément, y compris les grilles. Sur les blocs ci-dessous, nous ajoutons deux classes : ui-bar pour ajouter l'espacement de la barre par défaut et ui-bar-e pour appliquer le fond dégradé et le style de police pour le nuancier de thème "e" de la barre d'outils. A des fins d'illustration, un attribut d'alignement style="height:120px" est également ajouté à chaque grille pour mettre à chacun une hauteur standard.

Grilles trois colonnes

L'autre configuration de grille de mise en page utilise class=ui-grid-b sur le parent et 3 éléments conteneur enfant, chacun avec sa classe respective ui-block-a/b/c, pour créer une disposition en trois colonnes (33/33/33%). Remarque : ces blocs sont également stylisés avec des classes de thème afin que la grille de mise en page soit clairement visible.


<div class="ui-grid-b">
	<div class="ui-block-a">Bloc A</div>
	<div class="ui-block-b">Bloc B</div>
	<div class="ui-block-c">Bloc C</div>
</div><!-- /grid-b -->

Ceci va produire une grille 33/33/33% pour notre contenu.

Grilles à lignes multiples

Les grilles sont conçues pour envelopper de multiples lignes d'élément. Par exemple, si vous spécifiez une grille à 3 colonnes (ui-grid-b) sur un conteneur qui a neuf blocs enfant, il se terminera en 3 lignes de 3 éléments chacun. Il y a une règle CSS pour effacer les flotteurs et commencer une nouvelle ligne lorsque la class = ui-bloc-une est vu donc assurez-vous d'attribuer des classes de bloc dans une séquence répétée (a, b, c, a, b, c, etc) qui correspond au type de la grille :
