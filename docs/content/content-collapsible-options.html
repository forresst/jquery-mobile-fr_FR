Le plugin pliable a les options suivantes :

collapsed boolean

Par défaut : true

Lorsque c'est à false, le conteneur est élargi initialement avec un moins comme icône dans l'entête. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.collapsible.prototype.options.collapsed = false;
});

Cette option est également définie comme un attribut de données : data-collapsed="false".

collapseCueText string

Par défault : " click to collapse contents"

Ce texte est utilisé pour fournir des commentaires sonores pour les utilisateurs avec un logiciel de lecteur d'écran. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.collapsible.prototype.options.collapseCueText = " collapse with a click";
});
contentTheme string

par défaut : null, hérité d'un parent

Définit le jeu de couleurs (nuancier) spécifiquement pour le bloc de contenu du pliable. Il accepte une seule lettre de A à Z qui correspond aux nuanciers inclus dans votre thème. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.collapsible.prototype.options.contentTheme = "a";
});

Cette option est également définie comme un attribut de données : data-content-theme="a".

expandCueText string

Par défaut : " click to expand contents"

Ce texte est utilisé pour fournir des commentaires sonores pour les utilisateurs avec un logiciel de lecteur d'écran. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.collapsible.prototype.options.expandCueText = " expand with a click";
});
heading string

Par défaut : "h1,h2,h3,h4,h5,h6,legend"

Dans le conteneur du pliable, le premier élément enfant immédiat qui correspond à ce sélecteur sera utilisé comme l'entête pour le pliable. Pour changer les éléments qui sont initialisés, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.collapsible.prototype.options.heading = ".mycollapsibleheading";
});
iconpos string

par défaut : "left"

Positions de l'icône dans l'entête du pliable. Les valeurs possibles : left, right, top, bottom, none, notext.

$( document ).bind( "mobileinit", function(){
    $.mobile.collapsible.prototype.options.iconpos = "right";
});

Cette option est également définie comme un attribut de données : data-iconpos="right".

initSelector CSS selector string

Par défaut : ":jqmData(role='collapsible')"

Ceci est utilisé pour définir les sélecteurs (types d'éléments, les rôles de données, etc) qui seront automatiquement initialisé comme des pliables. Pour changer les éléments qui sont initialisés, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.collapsible.prototype.options.initSelector = ".mycollapsible";
});
mini boolean

par défaut : false

Définit la taille de l'élément pour qu'il soit plus compact : une version mini. Si la valeur est false pour un conteneur individuel de pliable, mais que ce conteneur fait partie d'un ensemble pliable, alors la valeur est héritée de l'ensemble du parent pliable. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.collapsible.prototype.options.mini = true;
});

Cette option est également définie comme un attribut de données : data-mini="true".

theme string

par défaut : null, hérité d'un parent

Définit le jeu de couleurs (nuancier) pour le pliable. Il accepte une seule lettre de A à Z qui correspond aux nuanciers inclus dans votre thème. Pour définir la valeur pour toutes les instances de ce widget, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
    $.mobile.collapsible.prototype.options.theme = "a";
});

Cette option est également définie comme un attribut de données : data-theme="a".
