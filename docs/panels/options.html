animate par défaut : true
Définit pour que le panneau s'anime à l'ouverture et à la fermeture. Si c'est à false, le panneau apparaîtra et disparaîtra sans animation. Cela est recommandé pour des raisons de performance. Également disponible via l'attribut data-animate sur le conteneur du panneau.
dismissible par défaut : true
Définit si le panneau peut être fermé en cliquant sur ​​l'extérieur sur la page. Également disponible via l'attribut data-dismissible sur le lien qui ouvre le panneau.
display par défaut : "reveal"
La relation entre le panneau et le contenu de la page. Peut être soit poussé sur la page ("reveal"), soit rétrécir le contenu pour s'adapter au contenu du panneau sous forme de colonne ("push"), ou soit superposer le contenu ("overlay"). Également disponible via l'attribut data-display sur le lien qui ouvre le panneau.
initSelector par défaut : ":jqmData(role='panel')"
Le rôle (et ou un sélecteur jQuery valide) utilisé pour déclencher l'auto-initialisation du widget panneau.
position par défaut : "left"
Le côté de l'écran, où le panneau apparaît. Les valeurs peuvent être "left" ou "right. Également disponible via l'attribut data-position sur le lien qui ouvre le panneau.
positionFixed default: false
Définit si le panneau a une position fixe, ainsi les contenus sont visibles, même si la page défile vers le bas. Cela permet également de faire défiler la page tandis que le panneau reste fixe. Nous vous recommandons de ne pas activer cette fonction lorsque les panneaux sont utilisés dans des applications Android pour des raisons de mauvaises performances et des problèmes d'affichage. Également disponible via l'attribut data-position-fixed sur le panneau.
swipeClose par défaut : "true"
Définit si le panneau peut être fermé en glissant au dessus du panneau, ou si dismissible=true, sur le contenu de la page et de la barre d'outils . Également disponible via l'attribut data-swipe-close sur le panneau.
theme par défaut : null
Nuancier du thème pour le panneau. Peut être une lettre de nuancier valable dans votre thème (a-z). Également disponible via l'attribut data-theme sur le conteneur du panneau.

L'option Classes

L'option classes définit les noms de classes structurelles que le plugin utilise. Ce n'est que configurable via JavaScript car il attend une valeur littérale d'objet.

classes.panel par défaut : "ui-panel"
Classe ajoutée au panneau.
classes.panelOpen par défaut : "ui-panel-open"
Classe ajoutée au panneau lors de l'ouverture. Utilisées pour cibler accélération matérielle uniquement pendant les transitions.
classes.panelClosed par défaut : "ui-panel-closed"
Classe ajoutée au panneau lors de la fermeture.
classes.panelFixed par défaut : "ui-panel-fixed"
Classe ajoutée au panneau lorsque le positionnement fixe est appliqué.
classes.panelInner par défaul : "ui-panel-inner"
Classe ajoutée au div qui enveloppe les contenus du panneau.
classes.modal par défaut : "ui-panel-dismiss"
Classe ajoutée à la superposition sur la page pour rejeter le panneau lorsqu'il est masqué.
classes.modalOpen par défaut : "ui-panel-dismiss-open"
Classe ajoutée à la couche invisible sur la page lorsque le panneau est ouvert. Utilisé pour rejeter (dismiss) le panneau.
classes.pagePanel par défaut : "ui-page-panel"
Classe ajoutée au conteneur de la page lorsque le widget panneau est présent.
classes.pagePanelOpen par défaut : "ui-page-panel-open"
Classe ajoutée à la page lorsqu'un panneau est ouvert.
classes.contentWrap par défaut : "ui-panel-content-wrap"
Classe ajoutée au conteneur injecté autour du contenu de la page (header, content, footer), nécessaire pour la transition du contenu de page. Les barres d'outils fixes ne sont pas enveloppées.
classes.contentWrapOpen par défaut : "ui-panel-content-wrap-open"
Classe ajoutée aux contenus de la page enveloppé lorsque le panneau est ouvert.
classes.contentWrapClosed par défaut : "ui-panel-content-wrap-closed"
Classe ajoutée au conteneur du contenu de la page lorsque l'animation de fermeture est terminée.
classes.contentFixedToolbar par défaut : "ui-panel-fixed-toolbar-wrap"
Classe ajoutée à l'entête et au pied de page fixe, nécessaire pour assurer la transition des barres d'outils fixes.
classes.contentFixedToolbarOpen par défaut : "ui-panel-content-fixed-toolbar-open"
Classe ajoutée aux barres d'outils fixes lorsque le panneau est ouvert.
classes.contentFixedToolbarClosed par défaut : "ui-panel-content-fixed-toolbar-closed"
Classe ajoutée aux barres d'outils fixes après que l'animation de fermeture est terminée.
classes.animate par défaut : "ui-panel-animate"
Classe ajoutée au panneau, au contenu de page enveloppé et aux barres d'outils fixes lorsque l'option animate est true et que le test de fonctionnalité de transformation 3D retourne true.
