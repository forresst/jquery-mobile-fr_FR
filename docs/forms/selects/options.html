Le plugin menu de sélection a les options suivantes :

corners boolean

Par défaut : true

Applique une bordure arrondi de bouton du thème au bouton de sélection si défini à true. Cette option est également définie comme un attribut de données : data-corners="false"

$('select').selectmenu({ corners: "false" });
icon string

default: "arrow-down"

Applique une icône de l'ensemble d'icônes au bouton personnalisé. Cette option est également définie comme un attribut de données : data-icon="star"

$('select').selectmenu({ icon: "star" });
iconpos string

default: "right"

Positions de l'icône dans le bouton de sélection. Les valeurs possibles : left, right, none, notext. La valeur notext affichera la sélection comme un bouton icône-seule sans texte. Cette option est également définie comme un attribut de données : data-iconpos="left"

$('select').selectmenu({ iconpos: "left" });
iconshadow boolean

Par défaut : true

Applique une ombre de thème au bouton de sélection si défini à true. Cette option est également définie comme un attribut de données : data-iconshadow="false"

$('select').selectmenu({ iconshadow: "false" });
initSelector CSS selector string

par défaut : "select:not(:jqmData(role='slider'))"

Ceci est utilisé pour définir les sélecteurs (types d'éléments, les data roles, etc) qui sera automatiquement initialisé comme les menus de sélection. Pour changer les éléments qui sont initialisés, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
	$.mobile.selectmenu.prototype.options.initSelector = ".myselect";
});
inline boolean

par défaut : null (false)

Si positionné à true, cela rendra le bouton sélection comme un bouton aligné pour que la largeur soit déterminée par le texte du bouton. Par défaut, c'est null (false) si le bouton de sélection est sur toute la largeur, indépendamment du contenu des commentaires. Valeurs possibles : true, false. Cette option est également définie comme un attribut de données : data-inline="true"

$('select').selectmenu({ inline: "true" });
mini boolean

par défaut : false

Définit la taille de l'élément pour qu'il soit plus compact : une version mini. Cette option est également définie comme un attribut de données : data-mini="true"

$('select').selectmenu({ mini: "true" });
nativeMenu boolean

Par défaut : true

Lorsqu'il est réglé à true, en cliquant sur le menu de sélection personnalisé, cela ouvrira le menu de sélection natif qui est mieux pour la performance. Si c'est à false, le style du menu de sélection personnalisé sera utilisé à la place du menu natif. Cette option est également exposée comme un attribut de données : data-native-menu="false"

$('select').selectmenu({ nativeMenu: "false" });
overlayTheme string

Par défaut : a

Définit la couleur de la couche de recouvrement pour la boîte de dialogue basée sur des menus de sélection et la bordure extérieure des petits menus personnalisés. Il accepte une seule lettre de A à Z qui correspond aux échantillons inclus dans votre thème. Par défaut, les couleurs de blocs de contenu pour l'échantillon A seront utilisés pour les superpositions. Cette option est également définie comme un attribut de données : ui-body-d

$('select').selectmenu({ overlayTheme: "d" });
preventFocusZoom boolean

Par défaut : true sur les plateformes iOS

Cette option désactive temporairement le zoom de la page quand une sélection personnalisée est actuellement utilisée, ce qui empêche les appareils iOS d'effectuer un zoom de la page dans la sélection. Par défaut, iOS fait souvent un zoom sur les contrôles de formulaire, et le comportement est souvent inutile et intrusif dans les mises en pages optimisés pour le mobile. Cette option est également définie comme un attribut de données : data-prevent-focus-zoom="true"

$('select').selectmenu({ preventFocusZoom: true });
shadow boolean

Par défaut : true

Applique le style d'ombre portée au bouton de sélection s'il est définit à true. Cette option est également définie comme un attribut de données : data-shadow="false"

$('select').selectmenu({ shadow: "false" });
theme string

par défaut : null, hérité d'un parent

Définit le jeu de couleurs (échantillon) pour toutes les instances de ce widget. Il accepte une seule lettre de A à Z qui correspond aux échantillons inclus dans votre thème. Par défaut, il hérite de la même couleur de l'échantillon que possède le conteneur parent s'il ne est pas explicitement défini. Cette option est également définie comme un attribut de données : data-theme="a"

$('select').selectmenu({ theme: "a" });
