Les champs de texte et les zones de textes sont codés avec les éléments HTML standard, puis renforcée par jQuery Mobile pour les rendre plus attractifs et utilisable sur un appareil mobile. Regardez la reference de data- attribute pour voir tous les attributs possibles que vous pouvez ajouter à des saisies de texte.

Textes de saisie

Pour collecter du texte alphanumérique standard, utilisez un input avec un attribut type="text". Réglez l'attribut for de label pour faire correspondre l'id de l'input ainsi ils sont sémantiquement liés. Il est possible de cacher le label d'accessibilité si il n'est pas souhaitée dans la mise en page, mais nous avons besoin qu'il soit présent dans le balisage pour des raisons de sémantique et d'accessibilité.

	
    <label for="basic">Saisie de texte :</label>
    <input type="text" name="name" id="basic" value=""  />

Cela va produire une saisie de texte de base. Les styles par défaut définissent la largeur du champ de saisie à 100% du conteneur parent et empile les labels sur une ligne distincte.

Version mini

Pour avoir une version plus compacte, qui est utile dans les barres d'outils et les espaces restreints, ajoutez l'attribut data-mini="true" à l'élément pour créer une version mini. 

	
<label for="basic">Texte de saisie :</label>
<input type="text" name="name" id="basic" value="" data-mini="true" />

Ceci va produire une saisie pas aussi grande que la version standard et disposant d'une taille inférieure.

Conteneurs de champs

Optionnellement, enveloppez-les dans un conteneur avec l'attribut data-role="fieldcontain" afin de les grouper visuellement dans un formulaire plus long.

	
<div data-role="fieldcontain">
    <label for="name">Saisie de texte :</label>
    <input type="text" name="name" id="name" value=""  />
</div>

Plus de types de saisie de texte

Dans jQuery Mobile, vous pouvez utiliser les types existants et nouveaux de saisie tels que password, email, tel, number, et bien d'autres Certaines valeurs de type sont rendus différemment selon les navigateurs. Par exemple, Chrome rend l'input range comme un curseur. jQuery Mobile standardise l'apparence de range et search dynamiquement en changeant leur type en text. Vous pouvez configurer les types d'entrée qui sont dégradés en text avec les options du plugin page.

Un avantage majeur de l'utilisation de ces types d'entrée plus spécifique si cela sur des appareils mobiles, claviers spécialisés que la vitesse de saisie de données sont offerts à la place du clavier de texte standard. Essayez les inputs suivants sur un appareil mobile pour voir, dans quel cas ils affichent les claviers personnalisés sur différentes plateformes.

Zones de saisie

Pour les entrées de texte multi-ligne, utiliser un élément textarea. The framework will auto-grow the height of the textarea to avoid the need for an internal scrollbar.

Réglez l'attribut for du label pour correspondre à l'id du textarea ainsi ils sont sémantiquement liés et enveloppez les dans un div avec l'attribut data-role="fieldcontain" pour les grouper.

	
<label for="textarea-a">Zone de saisie:</label>
<textarea name="textarea" id="textarea-a">
Je suis une zone de saisie basique. Si cela est pré-rempli avec du contenu, la hauteur sera automatiquement ajusté sans avoir à faire de scroll. C'est une caractéristique très pratique.
</textarea>

Ceci va produire une zone de saisie de base avec la largeur définie à 100% du conteneur parent et le label empilés sur un ligne séparée. La zone de saisie va croître pour s'adapter aux nouvelles lignes que vous ajouterez :

	
<div data-role="fieldcontain">
<label for="textarea">Zone de saisie :</label>
	<textarea name="textarea" id="textarea"></textarea>
</div>

Appel du plugin textinput

Ce plugin sera automatiquement initialiser sur n'importe quelle page qui contient une zone de saisie ou n'importe quel type de texte listé au dessus, pas besoin de l'attribut data-role dans le balisage. Cependant, si nécessaire, vous pouvez appeler directement le plugin textinput sur n'importe quel sélecteur, comme n'importe quel plugin jQuery :


$('input').textinput();

Types de saisie dégradés

jQuery Mobile dégrade plusieurs types d'entrée HTML5 pour revenir à type=text, ou type=number, après l'ajout de contrôles améliorés. Par exemple, les saisies avec un type range sont améliorés avec un contrôle de curseur personnalisé et leur type est mis à number pour offrir un saisie de formulaire utilisable à côté du curseur. Les saisies de type search sont dégradées vers type=text après que nous ayons ajouté nos propres styles thématisable de saisie de recherche.

Le plugin page contient une liste des types de saisie, qui sont mis à true ce qui signifie qu'ils vont se dégrader en type=text, false ce qui signifie qu'ils ne vont pas être modifiés, ou une chaîne telle que "number", ce qui signifie qu'ils vont être convertis dans ce type (comme le cas de type=range).

Vous pouvez configurer les types qui sont modifiés via l'option degradeInputs du plugin de la page, qui peut être manipulé en externe via$.mobile.page.prototype.options.degradeInputs, qui a les propriétés : color, date, datetime, "datetime-local", email, month, number, range, search, tel, time, url et week. Soyez sûr de configurer cela à l'intérieur d'un gestionnaire d'évènement lié à l'événement mobileinit, afin qu'il s'applique à la première page ainsi qu'aux pages suivantes qui seront chargées.
