Les zones de saisie de recherche sont un nouveau type HTML qui est stylisé en forme de pilule avec des coins et ajoute une icône "x" pour effacer le champ une fois que vous commencez à taper. Commencez avec un input dont l'attribut est type="search" dans votre balisage. Regardez la reference de data- attribute pour voir tous les attributs possibles que vous pouvez ajouter aux saisies de recherche.

Réglez l'attribut for de label pour faire correspondre l'ID de l'input ainsi ils sont sémantiquement liés. Il est possible de cacher le label d'accessibilité si il n'est pas souhaitée dans la mise en page, mais nous avons besoin qu'il soit présent dans le balisage pour des raisons de sémantique et d'accessibilité.

	
<label for="search-basic">Texte de recherche :</label>
<input type="search" name="search" id="searc-basic" value="" />

Cela va produire une saisie de recherche de base. Les styles par défaut définissent la largeur du champ de saisie à 100% du conteneur parent et empile les labels sur une ligne distincte.

Version mini

Pour avoir une version plus compacte, qui est utile dans les barres d'outils et les espaces restreints, ajoutez l'attribut data-mini="true" à l'élément pour créer une version mini. 

	
<label for="search-basic">Texte de recherche :</label>
<input type="search" name="search" id="searc-basic" value="" data-mini="true" />

Ceci va produire une entrée de recherche pas aussi grande que la version standard et disposant d'une taille inférieure.

Conteneurs de champs

Optionnellement, enveloppez-les dans un conteneur avec l'attribut data-role="fieldcontain" afin de les grouper visuellement dans un formulaire plus long.

	
<div data-role="fieldcontain">
    <label for="search">Recherche :</label>
    <input type="search" name="password" id="search" value="" />
</div>

Thématisation

L'attribut data-theme peut être ajouter à la saisie d'une recherche pour définir un thème avec une lettre de nuancier.

Paramétrage du texte du bouton de nettoyage

Le texte du bouton utilisé pour effacer l'entrée de recherche de texte peut être configuré pour toutes les entrées de recherche en liant l'événement mobileinit et en paramétrant la propriété $.mobile.textinput.prototype.options.clearSearchButtonText avec le texte de votre choix.

Appel du plugin textinput

Ce plugin sera auto-initialiser sur n'importe quelle page qui contient une saisie de texte avec l'attribut type="search", pas besoin d'un attribut data-role dans le balisage. Cependant, si nécessaire, vous pouvez appeler directement le plugin textinput sur un sélecteur, comme n'importe quel plugin jQuery :


$('.mySearchInput').textinput();
