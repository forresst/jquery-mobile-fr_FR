Pour créer un popup, ajoutez l'attribut data-role="popup" au div avec le contenu du popup. Ensuite créez un lien avec href défini avec l'id du div du popup et ajoutez l'attribut data-rel="popup" pour appeler le framework afin d'ouvrir le popup lorsqu'il sera touché. Il s'agit d'un modèle de balisage similaire au widget de la boîte de dialogue. Un div de popup doit être imbriqué à l'intérieur de la même page que le lien.


<a href="#popupBasic" data-rel="popup">Open Popup</a>

<div data-role="popup" id="popupBasic">
	<p>This is a completely basic popup, no options set.<p>
</div>

Ouvrir le Popup

Il s'agit d'un popup de base, sans options définies.

La popup se compose de deux éléments : l'écran, qui est un élément transparent ou translucide qui couvre l'ensemble du document, et le conteneur, qui est la popup. Si votre élément d'origine avait un attribut id, l'écran et le conteneur recevront chacun un attribut id basé sur celui-ci. L'id de l'écran sera suivi par "-screen" et l'id du conteneur avec "-popup" (dans l'exemple ci-dessus : id="popupBasic-screen" et id="popupBasic-popup").

Le framework ajoute de petite marge pour des éléments de texte, mais, c'est juste un conteneur avec des coins arrondis et une ombre qui sert d'ardoise vierge pour vos conceptions (même ces fonctions peuvent être désactivées via options) .

Ce style simple permet d'être ajouter facilement dans des widgets pour créer différentes interactions. Voici quelques exemples du monde réel qui combinent les différents paramètres et les styles que vous pouvez réaliser avec les popups:

Appel du plugin popup

Ce plugin sera auto-initialiser sur n'importe quelle page qui contient un div avec l'attribut data-role="popup". Cependant, si nécessaire, vous pouvez appeler directement le plugin popup sur n'importe quel sélecteur, comme n'importe quel plugin jQuery et travailler par programmation avec l'API du popup via les options, les méthodes et les événements :


$( "#myPopupDiv" ).popup();

Ouverture des popups

En utilisant la configuration basée sur le balisage, lorsqu'un lien avec data-rel="popup" est touché, le conteneur du popup correspondant à l'id référencé dans le href du lien sera affiché. Pour ouvrir un popup par programmation, appelez le popup avec la méthode open sur le conteneur du popup :


$( "#myPopupDiv" ).popup( "open" )

Fermeture des popups

Les popups peuvent être fermés soit en cliquant en dehors du widget popup, soit en pressant la touche Esc. Les popups peuvent aussi être fermés via la méthode close :


$( "#myPopupDiv" ).popup( "close" )

Pour ajouter un bouton fermer explicite au popup, ajoutez un lien avec le rôle button dans le conteneur du popup avec un attribut data-rel="back" qui fermera le popup lorsqu'il sera touché. Nous avons créer des classes d'aide pour positionner les boutons dans le coin supérieur gauche (ui-btn-left) ou le coin supérieur droit (ui-btn-right) du popup, mais vous devrez peut-être modifier ou ajouter ces styles de positionnement selon votre conception. Nous recommandons de'ajouter un espacement de contenu standard pour la popup pour faire de la place pour les boutons (voir la section suivante).


<div data-role="popup">
	<a href="#" data-rel="back" data-role="button" data-theme="a" data-icon="delete" data-iconpos="notext" class="ui-btn-right">Fermer</a>
	...le contenu du popup se place ici...
</div>

Ajout d'espacement

Pour les popups avec du texte, l'espacement est nécessaire. Nous recommandons l'ajout de la classe ui-content au conteneur du popup qui ajoutera l'espacement standard de 15px, comme le conteneur contenant une page. Si vous avez besoin, écrivez vos propres styles pour créer un design plus personnalisé.


<a href="#popupPadded" data-rel="popup" data-role="button">Popup avec espacement</a>

<div data-role="popup" id="popupPadded" class="ui-content">
	<p>C'est un popup avec la classe <code>ui-content</code> ajouté au conteneur du popup.</p>
</div>

Lorsque l'espacement est ajouté, nous appliquons quelques règles de style pour annuler la marge supérieure de l'entête ou d'un paragraphe dans le popup et faisons de même pour la marge du bas du dernier élément. Cela permet de éviter un espace vertical trop important les popups, lorsque l'espacement du contenu et les marges des éléments se combinent.

Options de positionnement

Par défaut, les popups s'ouvrent centrées verticalement et horizontalement sur ​​la chose que vous avez cliqué (l'origine) ce qui est bien pour les popups utilisées sous forme d'infobulles ou de menu. Le framework applique également quelques règles de base sur la détection de collision afin d'assurer que la fenêtre contextuelle apparaisse sur l'écran, ainsi la position finale peut ne pas être toujours centrée sur l'origine.

Dans le cas d'une boîte de dialogue ou d'un l'album, la popup doit apparaître centrée dans la fenêtre à la place de l'origine, il faut donc ajouter l'attribut data-position-to au lien et spécifier la valeur window.

Il est aussi possible de spécifier n'importe quel sélecteur valide comme valeur de position-to en plus de origin et window. Par exemple, vous pouvez définir data-position-to="#monElement" pour avoir une popup qui apparaisse sur l'élement avec l'id monElement.


<a href="#positionWindow" data-rel="popup" data-position-to="window">

<div data-role="popup" id="positionWindow">
	<p>Je suis positionné par rapport à la fenêtre.</p>
</div>

Les contraintes de placement de la popup, qui peuvent ne pas centrer comme vous le souhaitez la popup, sont les suivantes :

  1. La largeur de la popup sera limitée en utilisant le CSS max-width sur la largeur de la fenêtre moins une tolérance de part et d'autre de 15px.
  2. Une tolérance à partir des bords de la fenêtre (15px sur chacun des côtés et 30px sur le haut et le bas) sera observé lorsque la popup s'adapte à l'intérieur de la fenêtre. Les grandes popups peuvent déborder des bords supérieurs et inférieurs de la fenêtre. Ces parties de la popup peuvent être consultées en défilant manuellement le document. Cette tolérance est une option configurable
  3. La coordonnée supérieure de la popup ne sera jamais négative. Cela garantit que le haut de la popup ne soit pas coupée.
  4. Si le centrage de la popup sur un élément entraîne une augmentation de la hauteur totale du document, la popup est décalé vers le haut tout au plus jusqu'à ce que sa coordonnée supérieure soit à 0.

Notez également qu'une popup est actuellement toujours placée au centre de la fenêtre après un changement d'orientation ou d'un événement de redimensionnement de la fenêtre.

Regardez les méthodes pour plus d'informations sur le pramètrage du positionnement de la popup par programmation, en incluant l'option pour spécifier les coordonnées x et y.

Paramétrage des transitions

Par défaut, les popups n'ont pas de transition pour s'ouvrir aussi rapidement que possible. Pour définir la transition utilisée par la popup, ajoutez l'attribut data-transition au lien qui référence la popup. La version inversée de la transition sera utilisé lors de la fermeture de la popup.


<a href="#transitionExample" data-transition="flip" data-rel="popup">
   Transition Flip
</a>

Pour des raisons de performances sur les appareils mobiles, nous vous recommandons d'utiliser des transitions simples comme pop, fade ou non pour avoir des animations de popup fluide et rapide, en particulier avec de larges ou de complexes widgets dans un popup. Pour apercevoir tous les types de transition, vous devez être sur un navigateur qui supporte les transformations 3D. Par défaut, les dispositifs qui ne supportent pas la 3D (Comme Android 2.x), se replient sur "fade" pour tous les types de transition. Voir les transitions de pagepour plus d'informations sur le système de transition .

Thématisation du popup et de son fond

Le plugin popup fournit 2 options de thème : data-theme et data-overlay-theme. L'option data-theme fait référence au thème du popup, alors que data-overlay-theme se réfère au thème du fond du popup, qui couvre toute la fenêtre derrière le popup.

data-theme héritera de la page et aura toujours une valeur valide lorsque le popup s'ouvrira, à moins que vous spécifiez explicitement data-theme="none", dans ce cas, le popup aura un fond transparent.

Le data-overlay-theme ne sera jamais défini et le popup de fond, bien qu'il soit toujours présent lorsque le popup s'affiche, sera complètement transparent, à moins explicitement de le définir en utilisant par exemple data-overlay-theme="a". Dans ce cas, l'arrière-plan sera fade in, cachant partiellement le reste de la fenêtre, pour attirer l'attention sur le popup Voici un exemple d'un popup thématisé :


<div id="both" data-role="popup" data-theme="e" data-overlay-theme="a" class="ui-content">
  ...Contenus du popup...
</div>

Remarque : Le chainage des popups n'est pas permis

Le framework ne prend pas en charge le chaînage des popups car il n'est pas possible d'intégrer un lien d'un popup à l'intérieur d'un autre popup. Tous les liens avec data-rel="popup" à l'intérieur d'un popup ne fera rien du tout.

Cela signifie également que des menus de sélection ne fonctionneront pas à l'intérieur des popups, car ils sont eux-mêmes implémentés en utilisant des popups. Si vous placez un menu de sélection dans une popup, il sera rendu sous la forme d'un menu natif de sélection, même si vous spécifiez data-native-menu="false".

Une solution de contournement pour obtenir des popups enchaînées est l'utilisation d'une temporisation par exemple dans l'évènement popupafterclose lié à la fenêtre contextuelle invoquer. Dans l'exemple suivant, lorsque la première popup est fermée, la seconde sera ouverte par un appel retardé à la méthode open.


$( document ).on( "pageinit", function() {
    $( '.popupParent' ).on({
        popupafterclose: function() {
            setTimeout( function(){ $( '.popupChild' ).popup( 'open' ) }, 100 );
        }
    });
});
