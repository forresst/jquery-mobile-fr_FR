Le plugin popup a les options suivantes :

corners boolean

Par défaut : true

Définit s'il faut dessiner un popup avec des coins arrondis. Cette option est également définie comme un attribut de données : data-corners="true"

$('.selector').popup({ corners: true });
initSelector CSS selector string

default: ":jqmData(role='popup')"

Ceci est utilisé pour définir les sélecteurs (types d'éléments, les rôles de données, etc) qui seront automatiquement initialisé comme des popups. Pour changer les éléments qui sont initialisés, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
   $.mobile.popup.prototype.options.initSelector = ".mypopup";
});
overlayTheme string

par défaut : null

Définit le jeu de couleurs (nuancier) pour le fond du popup, qui recouvre entièrement la fenêtre. S'il n'est pas explicitement définie, le fond sera transparent.

$('.selector').popup({ overlayTheme: "a" });

Cette option est également définie comme un attribut de données : data-overlay-theme="a"

positionTo string

Par défaut : "origin"

Définit l'élément relatif sur ​​lequel la popup sera centrée. Il a les valeurs suivantes :

"origin" Lorsque la popup s'ouvre, elle se centre au dessus des coordonnées passées à l'appel open() (voir la page méthodes).
"window" Lorsque la popup s'ouvre, elle se centre dans la fenêtre.
Sélecteur jQuery Lorsque la popup s'ouvre, elle créé un objet jQuery basé sur le sélecteur et le centre au dessus. Le sélecteur est filtré pour les éléments qui sont visibles avec ":visible". Si le résultat est vide, la popup sera centrée dans la fenêtre.

shadow boolean

Par défaut : true

Définit s'il faut dessiner une ombre autour du popup. Cette option est également définie comme un attribut de données : data-shadow="true"

$('.selector').popup({ shadow: true });
theme string

par défaut : null

Définit le jeu de couleurs (nuancier) pour le contenu du popup. Sauf si il est explicitement défini à 'none', le thème pour le popup sera défini la première fois par l'héritage du thème de la page ou en cas d'échec par la valeur 'c' codé en dur. Si vous le définissez à 'none', le popup n'aura pas de thème du tout, et sera transparente.

$('.selector').popup({ theme: "a" });

Cette option est également définie comme un attribut de données : data-theme="a"

tolerance string

Par défaut : "15,30,15,30"

Définit la distance minimale entre le bord de la fenêtre et celui du bord de la popup. Par défaut, les valeurs ci-dessus seront utilisés respectivement sur la distance gauche, haut, droite et bas de la fenêtre.

Vous pouvez spécifier une valeur pour cette option par l'une des quatre façons suivantes :

  1. Une chaîne vide, null ou une autre valeur fausse. Cela entraînera la popup à retourner aux valeurs par défaut ci-dessus
  2. Un nombre seul Ce nombre sera utilisé pour les quatre bords.
  3. Deux nombres séparés par une virgule. Le premier nombre est utilisé pour les bords gauche et droit de la fenêtre et le second nombre est utilisé pour les bords supérieur et inférieur de la fenêtre.

  4. Quatre nombres séparés par des virgules. Le premier sera utilisé pour le bord gauche, le second pour le bord supérieur, le troisième pour le bord droit et le quatrième pour le bord inférieur.

transition string

Par défaut : "none"

Définit la transition par défaut pour la popup. La valeur par défaut se traduira par aucune transition.

Si la popup est ouverte à partir d'un lien et le lien a l'attribut data-transition de défini, la valeur qui y est spécifiée remplace la valeur de cette option au moment de l'ouverture de la popup sur le lien.
